2.0 冯诺依曼计算机历史

冯诺依曼计算机简称为EDVAC(Electronic Discrete Variable Automatic Computer)也叫普林斯顿体系结构（Princetionarchitecture），因为冯诺依曼是普林斯顿大学教授。

EDVAC是世界上第一台存储程序计算机(也有说法是电子数字式计算机)，成为所有现代计算机的原型和范本，它被建造于 1940年.。它是最早的大型二进制系统中央处理器计算机，而不是十进制系统。建造于19世纪40年代,EDVAC 设计于1944年,在这之前它被安装在美国军队的弹道学研究实验室，该实验室建造于1940年８月，位于马里兰州。

作为一个二进制序列的计算机，EDVAC用于加工数学的计算，连续的大致内存容量是5.5kb，EDVAC被用于作为数据的媒介存贮磁带，每天运行时间超过２=０个小时，EDVAC 于1960年被BRLESC（Ballistic Research Laboratories Electronic Scientific Computer）替换成更大的内存和更快的响应时间。

冯·诺依曼体制的主要思想包括：
（1）采用二进制代码形式表示信息（数据、指令）；
（2）采用存储程序工作方式（冯·诺依曼思想最核心的概念）；
（3）计算机硬件系统由五大部件（运算器、控制器、存储器、输入设备和输出设备）组成。

🪶按照冯诺依曼体质的思想，运算器是冯诺依曼计算机的核心，计算器就是用来进行复杂的计算，至于运行哪一种计算是加法、减法、乘法等等取决于控制器，告诉控制器要执行什么指令由它去进行控制。那么运算的中间结果，运算的初始值，运算的最终结果需要有一个容器进行盛放就是存储器。

2.1 冯诺依曼计算机的特点

  1. 计算机由五大部件组成（存储器、运算器、控制器、输入设备、输出设备）
  2. 指令和数据以同等地位存于存储器，可按地址寻址
  3. 指令和数据用二进制表示
  4. 指令由操作码和地址码组成
  5. 存储程序
  6. 以运算器为核心

2.2 现代计算机硬件框图

内存也被称为主存，而辅存指的是计算机的外部存储、例如，硬盘、U盘、软盘以及光盘等等

对于像冯诺依曼体系这种复杂的系统，需要有一个系统复杂性管理的方法（3’Y）

  • 层次化（Hierachy）：将被设计的系统划分为多个模块或子模块
  • 系统化（Modularity）：有明确定义的功能和接口
  • 规则性（regularity）：模块更容易被重用

这边的思想远远不止用于设计冯诺依曼体系，对于日常工作中各种框架的理解和设计意义也是非凡的。

2.3 计算机的工作步骤

2.3.1 编程举例

计算：ax^2 + bx + c 的值

第一种方案：直接计算

取 x 到运算器中

乘以 x 在运算器中

乘以 a 在运算器中

存 ax^2 到存储器中

取 b 到运算器中

乘以 x 在运算器中

加 ax^2 在运算器中

加 c 在运算器中

第二种方案：变形后计算 (ax+b)x+c

取 x 到运算器中

乘以 a 在运算器中

加 b 在运算器中

乘以 x 在运算器中

加 c 在运算器中

对于同一台计算机而言，第一种方案需要8条指令，第二种方案只需要5条指令，显然第二种方案是更快的，对于存储空间的要求也是更少的。

2.3.3 存储器的基本组成

主存储器由 存储体、MAR、MDR 组成：

存储体的组成与生活例子对应如下：

存储体 — 存储单元 — 存储元件

大楼 — 房间 — 床位

  • 存储单元：存放一串二进制代码
  • 存储字：存储单元中二进制代码的组合
  • 存储字长：存储单元中二进制代码的位数，每个存储单元赋予一个地址号

MAR：全称 memory data register，主存数据寄存器，MDR用来保存要被写入地址单元或者从地址单元读入的数据。

MDR：全称memory address register，主存地址寄存器，MAR用来保存数据被传输到的位置的地址或者数据来源位置的地址。

2.3.5 控制器的基本组成

控制器CU主要由IR寄存器、PC寄存器组成；完成一条指令需要有三步：PC取指令、IR分析指令、CU控制单元执行指令。

2.3.6 主机完成一条指令的过程

（1）以取指令为例

  • 先要做的是拿到取数指令，PC中存储了下一条要执行指令的地址

    • PC将指令的地址送给MAR

    • MAR再送给存储体

    • 存储体将具体的指令送到MDR中

    • 将取出来的指令送到IR进行分析

    • 将指令的操作码部分交给CU控制单元

  • 然后拿到具体的数值，所以传递具体数值的地址

    • 因为IR分析指令后，里面有操作码和地址码，操作码表明了是取数指令，已经告诉了CU。现在要把具体数值的取出来。IR吧数值的地址传递到MAR
    • MAR再传给存储体
    • 存储体将具体的数值取出来放到MDR中
    • MDR将数值传到ACC寄存器中

（2）以存数指令为例

  • PC中存储了下一条指令的地址
    • PC将指令的地址传给MAR
    • MAR将地址传给存储体M
    • 存储体M将具体的指令传给MDR
    • MDR将具体的指令传给IR进行分析
    • 将指令的操作码部分交给CU控制单元
  • 然后就要存具体的数值了
    • IR将要存的数值的具体地址传给MAR
    • MAR传给存储体
    • ACC寄存器将具体的数值传给MDR
    • MDR将数值传给存储体进行保存

（3）ax^2+bx+c程序的运行过程

  • 将程序通过输入设备送至计算机
  • 程序首地址 -> PC
  • 启动程序运行
  • 取指令 PC -> MAR -> M -> MDR -> IR， (PC)+1->PC
  • 分析指令 OP(IR) -> CU
  • 执行指令 Ad(IR) -> MAR -> M -> MDR -> ACC
