3.查找文件

3.1 查找根目录的node block

我们通过nat查看一下根目录对应的blk addr，

  • dump.f2fs -n 0~-1 f2fs.img：把nat的映射关系dump下来

发现根目录ino=3对应的blkaddr为4097，则真正的地址为4097 * 4096 = 16,781,312 = 0x1001000

根节点的node block必然为f2fs_inode类型，对应的结构体为：

结构体中i_addr即为对应的data block的起始位置。

  • i_addr[0]位于0x1001168位置，其值为0x1601
  • 得到的有效信息为根目录只需要一个data block就可以存放根目录所有信息
  • 唯一的一个data block起始位置为：0x1601 * 4096Byte = 0x1601000

3.2 根目录的data block找到system文件夹

我们知道这个data block里面存放的其实都是dentry，所以这个data block是文件夹对应的f2fs_dentry_block

其中一个f2fs_dentry_block可以存放NR_DENTRY_IN_BLOCK（214）个目录项，如果超过需要扩展data block，且需要在前面的node block中通过i_addr[1]指明下一个data block的地址。

每一个目录项对应的结构体为f2fs_dir_entry

结构体f2fs_dir_entry占用11个字节

前面的目录项分别为. 和 ..，system文件夹对应的结构体信息为

可以分析出如下结果：

  • hashcode为0x27e420bb
  • ino为0x04【可以自己通过ls -li查看，发现结果一致】
  • namelen为0x06【system总共6个字符，也就是6个字节，发现结果一致】
  • filetype为0x02

3.3 查找system目录的node block

我们知道了system的inode为4，再去dump_nat查看一下，

发现system目录ino=4对应的blkaddr为4099，则真正的地址为4099 * 4096 = 16,789,504 = 0x1003000

system目录为f2fs_inode类型，其其实地址为0x1003000，对应结构体f2fs_inode
