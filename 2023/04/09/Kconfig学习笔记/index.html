Kconfig学习笔记

Kconfig学习笔记

1.menuconfig配置界面

1、子菜单 —>

表示有子菜单，按下回车可以进入子菜单。

2、中括号 []

在每一个选项前都有个括号，有的是中括号，有的是尖括号，还有的是圆括号。

[] 表示该选项只有两种选项，中括号中要么是空，要么是“*”；

使用 “空格” 键进行切换

3、尖括号 <>

<>选择相应的配置时，有3种选择，它们代表的含义分别如下。

  • *：将该功能编译进内核。
  • 空：不将该功能编译进内核。
  • M：将该功能编译成可以在需要时动态插入到内核中的模块。

使用 “空格” 键进行切换

4、模块配置圆括号 ()

而圆括号的内容是要你在所提供的几个选项中选择一项。

2.Kconfig语法

2.1 Kconfig类型

配置选项的类型：”bool”/“tristate”/“string”/“hex”/“int”

每个config菜单项都要有类型定义

  • bool：布尔类型
  • tristate三态：内建、模块、移除
  • string：字符串
  • hex：十六进制
  • int：整型

config 关键字下面的这几行是配置项属性。属性里面描述了配置项的类型、输入提示、依赖关系、帮助信息和默认值等。

  • LOCALVERSION_AUTO的第一行表示LOCALVERSION_AUTO是bool类型，可以通过空格键 使能 或 禁止 LOCALVERSION_AUTO。
  • default y 表示 LOCALVERSION_AUTO的默认值为 y，所以上一行会默认选中

2.6 depends on 和 select

  • depend on 说明 USER_RETURN_NOTIFIER 依赖于 HAVE_USER_RETURN_NOTIFIER。也就是说 HAVE_USER_RETURN_NOTIFIER选中以后 USER_RETURN_NOTIFIER 才能被选中
  • select 表示方向依赖，当选中 GPIO_ALTERA 以后，GPIO_GENERIC、GPIOLIB_IRQCHIP 这两个选中也会被选中

2.7 choice/endchoice

choice/endchoice 代码段定义了一组可选择项，将多个类似的配置项组合在一起，供用户单选或者多选。上面的类型就是让用户选择处理器架构，可以从 ARCH_MULTIPLATFORM、ARCH_REALVIEW中进行选择。
