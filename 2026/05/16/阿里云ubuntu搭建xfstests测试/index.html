7.quick组&auto组&all

xfstests 的 group 本质上是给测试用例打标签。一个测试可以同时属于多个组，例如 generic/001 可能既属于 auto，也属于 quick。官方文档说明，运行测试组用 ./check -g <group>，具体分组可以看源码里的 tests/*/group.list。

3. all 组

all 是 所有测试组。

含义：

  • 尽可能运行所有测试；
  • 包括 auto 之外的测试；
  • 也可能包括 dangerous 测试；
  • 可能触发 kernel panic、hang、长时间运行、极端压力场景；
  • 不建议在普通云主机或生产环境直接跑。

4. generic/563：cgroup writeback 记账失败，和 exFAT 数据一致性关系不大

失败片段：

这个测试的目的，是验证 cgroup aware writeback 能否正确统计 I/O。它会在不同 cgroup 里读写文件，然后检查 io.stat 里的 read/write bytes 是否落在预期范围内。

你的失败意思是：某个阶段预期应该统计到大约 16MB write，但是实际统计到的是 0。

这类失败通常是：

  1. cgroup v2 I/O accounting 环境问题；
  2. loop 设备上的 writeback 统计不稳定；
  3. 当前内核/块设备/loop/exFAT 组合不符合测试预期；
  4. 不是文件内容损坏，也不是 bitmap 泄漏，也不是 exFAT cache panic。
