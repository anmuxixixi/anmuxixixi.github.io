第四步：使用 Smatch 运行检测

Linux 内核构建系统原生支持使用外部检查工具。你可以通过给 make 传递 C= 和 CHECK= 参数来调用 Smatch。

参数说明：

  • C=1：只对需要重新编译的 C 文件运行检查。
  • C=2：强制对所有被构建的 C 文件运行检查。
  • CHECK="..."：指定要运行的检查器路径及其参数。对于内核，必须加上 -p=kernel 参数，以便 Smatch 加载内核专用的检查规则。
