WSL如何识别U盘

在 Windows上安装usbipd-win

Windows 默认不会将硬件设备直接映射给 WSL，你需要使用微软推荐的开源工具 usbipd-win。

  1. 在 Windows 上安装 usbipd-win： 前往 GitHub Releases 下载 .msi 并安装。
  2. 绑定设备（在管理员权限的 PowerShell 中执行）：
    • 查看列表：usbipd list（找到你 U 盘的 BUSID，例如 2-3）。
    • 绑定驱动：usbipd bind --busid 2-3（仅需执行一次）。
    • 连接到 WSL：usbipd attach --wsl --busid 2-3。

第一步比较顺利，但是绑定的时候就报错了

升级到最新的 WSL 预发行版

这个报错社区也有其他人遇到了：https://github.com/dorssel/usbipd-win/issues/1010

1️⃣第一阶段：在 WSL 里编译新内核

打开你的 WSL 终端（Ubuntu），依次执行以下命令：

1. 安装编译工具和依赖项

Bash

2. 下载微软官方的 WSL 内核源码（由于源码较大，这一步可能需要几分钟）

Bash

3. 应用微软的默认配置，并进入图形化配置界面

Bash

4. 按照 Issue 1010 的要求勾选驱动 此时你的终端会变成一个蓝底的菜单界面（可以用键盘的上下键移动，回车进入，按 Y 键勾选，勾选后前面会变成 [*] 或 <*>）：

  1. 找到并进入 Device Drivers --->
  2. 向下滚动，找到并进入 USB support --->
  3. 找到 Support for Host-side USB，按下键盘的 Y 键（确保它前面有个星号 *）。
  4. 向下找，找到 USB Mass Storage support，按 Y 键。并且把它的子选项也尽量都按 Y 选中。
  5. 找到 USB/IP support --->，按回车进入。
  6. 找到 VHCI hcd，按下键盘的 Y 键。
  7. 一直按键盘右下角的左右键选中 < Save > 保存，文件名默认 .config 按回车，然后一直选 < Exit > 退出菜单。

5. 开始编译内核

Bash

(去喝杯水，根据你电脑的性能，这大概需要 5 到 15 分钟。只要最后没有报 Error 就算成功。)

6. 把编译好的内核拿出来 编译完成后，新内核文件躲在深处。我们把它复制到你 Windows 的 C 盘用户目录下（我看你之前的终端路径是 Administrator，所以放在这里）：

Bash

2️⃣第二阶段：让 Windows 换上新内核

内核编译好后，WSL 的任务就结束了。现在回到 Windows 系统。

1. 创建配置文件 在你的 Windows 资源管理器里，进入 C:\Users\Administrator\ 目录。 新建一个文本文档，命名为 .wslconfig（注意前面有个点，如果提示改变扩展名点“是”）。

2. 填写内核路径 用记事本打开 .wslconfig，把下面这两行填进去并保存：

Ini, TOML

3. 重启 WSL 打开你的 Windows PowerShell，执行：
