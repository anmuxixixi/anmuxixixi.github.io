git复杂场景: fork子仓版本老PR提交后合入冲突

背景：因为之前就关注过exfatprogs，所以很早之前就fork一份出来。然后我提了一个PR到主仓。maintainer在合入的时候发现merge conflict了，让我重新提一下。

但是我又不想重新开一个PR，又想基于最新exfatprogs提交呢？

我给gpt的描述：我现在的问题是，我在提交pull request的时候拉取了一个老的分支，然后无法合并，我不想重新提交PR，如何更新仓库后重新提交一个git commit.

1. 添加 upstream（如果你之前没加过）

如果没有 upstream，你只能从自己的 fork (origin) 获取代码，无法直接抓取上游最新提交。为了同步upstream代码，所以需要git remote add xxxx

upstream 是一个 自定义的远程仓库名字，通常代表：你 fork 的源头（原始项目），也就是你要跟随更新的上游仓库

2. 获取 upstream 最新代码

3. 切到你的 PR 分支

4. 用 rebase 更新你的分支（关键步骤）

如果中间有冲突，按提示修复，然后继续：

直到 rebase 完成。

5. 强制推送回你的 fork（PR 会自动更新）

⚠️ 由于是 rebase，一定要 --force
