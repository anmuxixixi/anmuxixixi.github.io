1.4 接收并解析 CSW（设备响应处理）

CSW 是设备对 CBW 命令的 “执行结果反馈”，代码分 3 步处理：接收 CSW→重试异常→解析状态。

1. 接收 CSW（Bulk-In 端点）

  • 通过 us->recv_bulk_pipe（Bulk-In 端点）接收 13 字节 CSW（协议规定长度）；
  • cswlen：记录实际接收长度（应对设备返回 0 字节等异常）。

2. 异常重试（兼容设备 bug）

  • 场景 1：部分设备会在数据传输后添加多余的零长度包，导致第一次接收 CSW 时返回 0 字节，需重试；
  • 场景 2：端点 STALL 表示设备无法处理命令（如命令不支持），重试一次接收 CSW（部分设备重试后会恢复）；
  • 若重试后仍失败，返回传输错误。

3. CSW 解析（核心：验证 + 状态判断）

关键校验逻辑：

  • 标签校验：bcs->Tag == us->tag 确保 CSW 是当前 CBW 的响应（避免多命令并发时乱序）；
  • 签名校验：兼容部分设备的非标准签名（协议规定 CSW 签名为 0x53425355，部分设备用自定义签名）；
  • 状态码校验：bcs->Status 只能是 0/1/2（超出则为无效响应）。

4. 残留字节数处理（修正数据传输状态）

  • 背景：部分设备会报告虚假残留字节数（如命令成功但残留非 0），代码通过 “启发式检测” 标记这类设备，后续忽略残留值；
  • scsi_set_resid(srb, …)：向上层 SCSI 层反馈实际未传输的字节数（如读 1024 字节，残留 512 字节表示仅成功读 512 字节）。
