2.2.1.3 秘钥安装到内核keyring中

安装到keyring中

将秘钥安装到内核f2fs文件系统super_block的秘钥环keyring(s_master_keys)里面

❓下面来讲讲为什么要设计installProvisioningKey

目的就是为了备份秘钥，必要时恢复reload

  • 内核里真正用来解密文件的 key 最终会装到 filesystem keyring（挂载点私有）。
  • 但这个 key 是不可持久保存的：
    • 文件系统卸载、进程重启时，FS keyring 里的 key 会消失。
    • 系统可能需要在挂载后重新“reload”密钥。
  • 因此 Android 在 session keyring 里再放一份 "fscrypt-provisioning" key。
    • 这样下次需要时，可以通过 ref 找到这份 provisioning key，重新调用 FS_IOC_ADD_ENCRYPTION_KEY 安装到 fs keyring。

换句话说，fscrypt-provisioning key 就像一个“密钥缓存”，保证后续恢复可用

官方注释也写了

❓Q：神秘的ref到底是什么？

✅A：其实就是identifier，就是keyring的标识符。有了这个标识符，在keyring的众多秘钥里面就可以通过这个标识符找到正确的秘钥

3.f2fs模块

3.1 添加秘钥

来自章节《 秘钥安装到内核keyring中》，从用户态传递到内核的时候就一个秘钥raw和秘钥长度raw_size，其他啥也没初始化

函数 do_add_master_key 中，观察参数列表：

  • sb ：和分区有关，意味这这些 key 只允许用于所指分区；
  • secret：记录着 vold 传递过来的软加密秘钥【也就是一直提到的生成的随机值代替秘钥】
  • mk_spec: 记录着 Master Key 的 master_key_identifier

总结，在执行完 do_add_master_key 函数后：

  • 可以通过 master_key_identifier 从分区 super block 结构中 s_master_keys 指向的 keyring 【只针对f2fs本身，因为多用户场景下，f2fs本身的s_master_keys 指向的 keyring有好几把秘钥】中找到 Master Key；
