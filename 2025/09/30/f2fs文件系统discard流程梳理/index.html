f2fs文件系统discard流程梳理

🎗️首先，我们来讲一讲discard的意义，当用户态大量的写入删除操作后，磁盘势必碎片化很严重，此时会触发f2fs gc流程，对segment做整理，但这个整理只是文件系统内部的动作，ufs是不知道哪些是无效的，此时就要用到discard命令，经过block层封装到trim指令后下发到ufs器件，告诉器件这些block无效了，你可以GC（器件GC）！

1.discard cmd生产主流程

我们在 f2fs文件系统gc垃圾回收（一）概述 结尾的时候写道：gc搬运完以后，那些零零散散的segment此时处于prefree的状态，等道checkpoint的时候唤醒discard线程去清理！

在cp的过程中会扫描prefree的segment生成discard cmd

4.2 准备scsi命令

根据设备支持情况生成discard对应的具体指令：

provisioning_mode 表示这个磁盘支持的 逻辑块释放方式（LBP = Logical Block Provisioning）。

可能的模式有：

  • SD_LBP_UNMAP → 使用 SCSI UNMAP 命令
  • SD_LBP_WS16 → 使用 WRITE SAME(16) + UNMAP
  • SD_LBP_WS10 → 使用 WRITE SAME(10) + UNMAP
  • SD_LBP_ZERO → 使用 WRITE SAME(10) 填 0
  • default → 不支持 → 返回错误
