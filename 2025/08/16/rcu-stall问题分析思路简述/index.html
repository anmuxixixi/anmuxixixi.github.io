什么是rcu stall？

RCU (Read-Copy-Update) stall 问题通常发生在 RCU 回调函数没有在预期的时间内被执行，导致内核检测到回调延迟并报告警告。rcu stall timeout的时间默认是21秒，可以在内核的.config中查看:

为了能在rcu stall时得到更多信息，建议在开发版本中打开panic_on_rcu_stall，panic时如果有fulldump或vmcore生成，对定位问题有一定帮助； 正式版本建议关闭panic_on_rcu_stall，避免影响扩大化。

关注点2：rcu:(t=5250 jiffies g=6285 q=69)等

  • t=5250 ：rcu stall的超时时间是5250个jiffies，查看CONFIG_HZ=250，所以是21秒，和CONFIG_RCU_CPU_STALL_TIMEOUT=21是对应的上的；
  • g=6285：Grace-period sequence，标明当前是第多少个宽限期，没什么帮助；
  • q=69：未执行的rcu callback数量，没什么帮助；
  • Task dump for CPU 2: swapper/2 R running task 0010x0000002a ：dump当前rcu stall的cpu上running的task和调用栈。这个很关键，一般rcu stall的问题都是某高优先级task长时间占用cpu导致的。当前问题的调用栈很长，也很值得讨论下：
