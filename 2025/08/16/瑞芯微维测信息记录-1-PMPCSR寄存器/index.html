PMPCSR寄存器介绍

PMPCSR 寄存器（Program Counter Monitor Control and Status Register）是 Rockchip 平台（尤其是基于 ARM 架构的处理器）中与 CPU 程序计数器（PC）监控相关的硬件寄存器，主要用于在系统异常（如 panic、死机）时捕捉 CPU 当前的执行地址和运行状态，是调试多 CPU 系统故障的重要硬件支持。
以下是对 PMPCSR 寄存器的详细解析：

1. 核心功能

PMPCSR 的核心作用是实时记录 CPU 当前执行的指令地址（即程序计数器 PC 的值），并附带 CPU 的运行状态信息（如异常级别、安全状态等）。在系统发生崩溃（如 panic）时，内核可以通过读取该寄存器获取崩溃瞬间的 CPU 执行位置，为定位故障原因（如死锁、非法指令、访问越界等）提供关键依据。

尤其在多 CPU 系统中，当某个 CPU 陷入 “卡死” 状态时，PMPCSR 寄存器中存储的 PC 值可以反映该 CPU 最后执行的指令，帮助开发者判断其是否停留在某个循环、中断处理或锁等待中。

2. 寄存器结构与字段解析

PMPCSR 通常是一个 64 位寄存器（部分平台可能分高低 32 位寄存器，如代码中通过 PMPCSR_LO 和 PMPCSR_HI 分别读取低 32 位和高 32 位，再组合为 64 位值）。其关键字段如下（基于 Rockchip 平台常见定义，具体可能因芯片型号略有差异）：

位域范围 含义 说明
63 安全状态位（NS/S） 1 表示非安全世界（NS，Non-Secure）；0 表示安全世界（S，Secure）。
61-63 异常级别（EL） 00：EL0（用户态）；01：EL1（内核态）；10：EL2（虚拟化层）；11：EL3（安全监控）。
0-60 程序计数器（PC）值 存储 CPU 当前执行的指令地址（低 61 位，具体有效位取决于异常级别）。

3. 适用场景

PMPCSR 主要用于系统故障调试，尤其是：

  • 系统 panic 时，快速定位各 CPU 崩溃前的执行地址。
  • 多 CPU 死锁排查，判断某个 CPU 是否停留在临界区或锁等待中。
  • 内核异常（如 OOPS、宕机）的现场信息收集，辅助分析故障栈。
