overlay中creds切换(chatgpt版)

✅ Android 12 内核中真正调用 override_creds() 的位置

Android 12 使用的 OverlayFS 特性引入了 mount 时强制 cred 切换（override_creds）。
adb remount → fs_mgr → 内核 overlayfs mount → 会进入这些调用点。

下面是你要查的 全部实际 override_creds 代码处。

🔥 ① fs/overlayfs/super.c — ovl_override_creds()

📌 AOSP Android 12 内核中最关键的 override_creds 调用点

函数：ovl_override_creds()

（Android 12 common kernel 中大约在 line ~ 1100–1200）

配套：ovl_revert_creds()

作用

OverlayFS 在某些操作（stat/lookup）会切换到 挂载者的 cred（creator_cred），
导致：

同一个 stat() 期间出现多个不同的 current_cred()

你看到的现象正是由这个函数导致的。

🔥 ② fs/overlayfs/file.c — 在 file read/write 前后切 cred

函数：ovl_read_iter() / ovl_write_iter()

作用：
OverlayFS 上读写文件时，临时切换 credential。

🔥 ③ fs/overlayfs/inode.c — getattr / lookup 会切 cred

函数：ovl_getattr()

关键点

stat() 就会走这里，因此你必定看到 current_cred 切两次。

这就是你观察到的根本来源。

🔥 ④ fs/overlayfs/namei.c — lookup 时临时切 cred

函数：ovl_lookup()

路径解析时（stat 的第一阶段）会调用 → 触发一次 cred 切换。

🔥 ⑤ fs/overlayfs/copy_up.c — copy-up 时也会 override_creds

例如 ovl_copy_up_one()：

虽然 stat 通常不会 copy-up，但逻辑上这里也会切 cred。

📌 总结：Android 12 中 override_creds 的完整链

文件 函数 场景
fs/overlayfs/super.c ovl_override_creds / ovl_revert_creds 所有 overlayfs 操作的核心切换
fs/overlayfs/inode.c ovl_getattr stat() 发生的 cred 切换之一
fs/overlayfs/namei.c ovl_lookup stat 路径解析阶段切 cred
fs/overlayfs/file.c ovl_read_iter / write_iter 读写文件
fs/overlayfs/copy_up.c ovl_copy_up_one copy-up 行为

🧠 为什么你看到两次不同的 current_cred？

因为 stat() 经过两个阶段：

阶段 内核函数 是否使用 override_creds
① path lookup ovl_lookup() ✔ credentials = creator_cred
② real getattr ovl_getattr() ✔ credentials = creator_cred

但是在某些路径（尤其 adb remount）：

  • 第一次 getattr 发生在 override_creds 期间
  • 第二次发生在 revert_creds 之后

所以你看到：

stat 调用期间 → LSM（selinux_inode_getattr）被调用两次

一次 cred = creator_cred
一次 cred = shell / system_server / adbd 的实际 cred

完美符合你的日志。
