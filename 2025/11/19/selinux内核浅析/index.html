1.基础概念

1.1 domain和type

当前有一条selinux策略：

位置 内容 角色 解释
第 1 个标识符 init domain（进程 type） 作用在主体（subject），即进程
第 2 个标识符 fsck_exec type（对象的标签） 作用在客体（object），即文件
第 3 个标识符 file class（对象类型） 被访问的对象属于 file 类
花括号权限 { read write } permissions 对 file class 的权限

什么是domian域？

SELinux 源自 Flask/Security-Enhanced Linux 的 Type Enforcement 模型。在 Type Enforcement 中：

  • Subject（进程）属于 domain
  • Object（文件等）属于 type

需要注意：一个进程永远只有一个 domain

1.2 domain和scontext的关系

我们经常在日志中看到avc denied，例如：audit(0.0:67): avc: denied { write }for path=”/dev/block/vold/93:96” dev=”tmpfs” ino=1263 scontext=u:r:kernel:s0 tcontext=u:object_r:block_device:s0 tclass=blk_file permissive=0

我们配规则的时候肯定这样配：allow kernel block_device:blk_file { write };

根据上面的知识：

  • kernel是进程domain，是进程的type
  • block_device是type，是对象的标签

❓如何查询每个标签的attribute呢？

  • 查看domaind的attribute：sesearch -A -s <domain> policy_file，例如：sesearch -A -s init policy
  • 查看type的attribute：sesearch -A -t <type> policy_file，例如：sesearch -A -t fsck_exec policy
  • [注]：policy_file通过可以通过adb pull /sys/fs/selinux/policy拉下来
