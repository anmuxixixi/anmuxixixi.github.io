CBW包

CBW （Command Block Wrapper，命令块包）是 USB 大容量存储设备类 (Mass Storage Class) 中 Bulk-Only Transport (BOT) 协议的关键数据结构，它具有以下特点：

  1. 定义与功能：
    • CBW 是主机 (Host) 向 USB 存储设备发送命令的载体，包含 SCSI 命令及其相关参数
    • 每个 CBW 固定为31 字节长度，不足需补 0
    • 通过Bulk-Out 端点进行传输，采用小端字节序 (低字节在前)
  2. CBW 的核心作用：
    • 封装 SCSI 命令 (CBWCB 字段)，指定设备执行的操作 (如读、写、查询等)
    • 包含数据传输方向和长度信息，指示后续数据传输的方式
    • 设备接收到 CBW 后执行相应操作，完成后通过 CSW (Command Status Wrapper) 返回状态

4-7 dCBWTag 60 39 88 43 主机标签（大端序为0x43883960），用于匹配后续 CSW
8-11 dCBWDataTransferLength 00 20 00 00 期望传输数据长度（大端序0x2000）→8192 字节
12 bmCBWFlags 80 数据方向：bit7=1 → 设备到主机（IN 方向）
13 bCBWLUN 00 逻辑单元号（LUN）→ 0 号设备（默认单设备 LUN）
14 bCBWCBLength 0A SCSI 命令（CBWCB）长度 → 10 字节
15-24 CBWCB（SCSI 命令） 28 00 00 14 8F 00 00 00 10 00 操作码28h（对应 SCSI 的READ (10) 命令）
25-30 填充字节 00…00 CBW 固定 31 字节，不足部分补 0

CSW包

CSW（Command Status Wrapper，命令状态包）是 USB 大容量存储设备类（Mass Storage Class）Bulk-Only Transport（BOT）协议中，设备对主机 CBW 命令的回应包，用于反馈命令的执行结果

查询磁盘信息命令块

Inquiry Command

Inquiry Data Format

  1. 识别设备类型：判断设备是磁盘（DIRECT-ACCESS）、光盘（CD-ROM）、磁带（SEQUENTIAL-ACCESS）还是打印机等 SCSI 支持的设备，决定后续使用何种驱动逻辑（如存储设备走块设备驱动，光盘走光驱驱动）。
  2. 确认设备厂商 / 型号 / 固件版本：获取厂商 ID、产品名称、固件版本号，用于驱动匹配（如特定厂商设备的兼容性适配）、故障排查（如固件版本相关 bug 定位）。
  3. 了解设备支持的 SCSI 协议版本：判断设备支持 SCSI-1、SCSI-2、SBC（SCSI Block Commands）等哪个版本，主机据此选择兼容的命令集（如是否使用现代 SBC-4 协议的高级功能）。
  4. 获取设备能力标识：例如是否支持同步传输、是否可移除（如 U 盘）、是否支持写保护、是否支持缓存等，主机据此优化 I/O 策略（如可移除设备需处理热插拔）。
